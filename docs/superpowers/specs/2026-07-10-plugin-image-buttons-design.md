# 图片生成插件按钮设计

## 范围

仅调整 `plugin-service/plugins/image-generation/web/index.html` 中图片生成插件页面的按钮视觉与交互反馈。主站全局 `.btn`、插件业务逻辑、接口和文案均不变。

## 设计方向

采用已确认的 A「轻量材质」方案，并保留项目现有青绿色品牌色。按钮使用克制的半透明表面、细边框、内侧高光和短阴影表达层级，不使用大面积渐变或夸张悬浮效果。

- 主要操作：青绿色实体材质，白色图标或文字，阴影轻且靠近表面。
- 次要操作：半透明浅色材质，配合 `backdrop-filter`；不支持模糊时仍有实色背景。
- 危险操作：浅红材质与红色文字；确认按钮可使用红色实体材质。
- 图标按钮：保持正方形触控区域，图标视觉居中，不使用文字胶囊替代熟悉图标。
- 圆角：常规按钮 8px，紧凑图标按钮 8px；已有抽屉边缘把手保留与边缘相连的形状。

## 交互状态

- 按下反馈在 `pointer-down` 对应的 `:active` 状态立即出现，使用 `transform: scale(0.97)`，约 80-100ms。
- 悬浮只轻微改变背景和阴影，不位移按钮，避免工具栏抖动。
- 键盘使用 `:focus-visible` 显示 2px 品牌色焦点环，鼠标点击不保留焦点环。
- 禁用状态不响应缩放，降低不透明度并保持可读性。
- `prefers-reduced-motion: reduce` 下移除按钮过渡和缩放。

## 移动端

- `max-width: 767px` 下，主要按钮和抽屉开关的触控区域至少为 44x44px。
- 历史记录删除控件至少 44px 高；通过扩大可点击区域保证触控，而不是放大文字。
- 弹窗底部操作在窄屏保持清晰间距；空间不足时按钮等宽排列，不允许文字溢出。
- 编辑器发送按钮从当前 40x40px 提升到 44x44px；工具控件允许换行且不挤压主操作。
- 适配安全区，不改变现有 `100dvh`、抽屉和滚动策略。

## 实现边界

新增样式放在插件 `index.html` 的现有 `<style>` 尾部，并以 `#app`、插件对话框类和 `data-testid` 选择器限定作用域。无需修改压缩后的 `assets/app.css`，也不引入 JavaScript 动画库。

## 验证

- 运行 `go test ./...` 验证插件服务行为没有回归。
- 检查桌面宽度 1440px，以及手机宽度 390px 和 360px。
- 覆盖主要、次要、删除、发送、抽屉开关、弹窗确认与取消按钮的默认、悬浮、按下、焦点和禁用状态。
- 检查浅色、深色和减少动态效果偏好。
- 确认按钮文字不溢出、控件不重叠、页面滚动与历史抽屉仍可操作。
