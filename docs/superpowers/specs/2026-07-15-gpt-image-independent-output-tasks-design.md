# GPT 多图独立任务设计

## 目标

GPT 模型生成多张图片或多个角度时，每张图片作为一个独立任务。任意任务成功后立即进入当前会话和历史记录，不等待其他任务完成；后续任务失败、取消或超时，不影响已经成功的图片。

## 范围

- 仅修改 `plugin-service/plugins/image-generation`。
- 适用于 GPT/OpenAI-compatible 同步生图模型的多图和多角度生成。
- Gemini batch 路径保持现状。
- 不修改主站 `backend` 服务和网关协议。

## 方案

前端在提交时把多图请求拆成多个现有的 `generate` 请求。普通多图复用同一提示词；多角度请求为每个角度使用对应的独立提示词，并固定 `output_count: 1`。每个请求由插件后端创建独立历史记录和本地任务，因此无需新增聚合任务或流式接口。

前端为每个子任务创建独立的待处理消息并保存其任务 ID。子任务完成时只替换自身的待处理消息，图片随即显示；子任务失败时只将自身标记为失败，其他任务继续运行。会话的全局生成状态在至少一个子任务运行时保持忙碌，所有任务进入终态后恢复空闲。

## 交互与状态

- 每张图片有独立的生成中、成功、失败或已停止状态。
- 成功图片立即显示，并对应一条独立历史记录。
- 失败项显示角度标签或图片序号及错误信息，可使用原参数单独重试。
- 用户停止生成时，取消所有仍在运行的子任务；已完成图片保持可见且不删除。
- 同一批提交中的完成顺序由实际上游返回顺序决定，原始角度/序号通过标签保留。

## 错误处理

单个请求失败不抛出整批失败，也不覆盖其他子任务的结果。只有全部子任务失败时才在会话级错误区域显示失败摘要；部分成功时保留成功结果，并在失败项上显示局部错误。

## 测试

- 多角度提交会发起多个 `output_count: 1` 请求，且提示词和标签正确。
- 第二个任务失败时，第一个任务的成功图片仍显示。
- 较早完成的任务无需等待较慢任务即可更新会话。
- 停止生成只取消未完成任务并保留成功结果。
- 普通 GPT 多图同样拆分为独立任务。
- 单图和 Gemini batch 行为不回归。
