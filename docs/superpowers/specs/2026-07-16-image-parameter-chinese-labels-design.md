# 图片生成参数中文标签设计

## 目标

将 GPT 图片生成参数中的画质、背景和参考图保真度显示为易懂的中文，同时保持组件状态、事件值、历史记录数据和 API 请求继续使用后端原始枚举。

## 映射

| 参数 | 原始值 | 中文标签 |
| --- | --- | --- |
| 画质 | `auto` | `自动画质` |
| 画质 | `low` | `低画质` |
| 画质 | `medium` | `标准画质` |
| 画质 | `high` | `高画质` |
| 背景 | `auto` | `自动背景` |
| 背景 | `transparent` | `透明背景` |
| 背景 | `opaque` | `不透明背景` |
| 保真度 | `low` | `低保真` |
| 保真度 | `high` | `高保真` |

后端将来返回未识别枚举时，界面原样显示该值，避免空白选项。

## 结构

新增一个无状态的共享格式化模块，按参数类型和原始值返回显示标签。顶部参数下拉框、新发送消息摘要和历史消息摘要共同调用该模块，避免重复映射。

原生 `option.value`、组件 `update:*` 事件、组合式状态和生成请求载荷不做转换。中文标签仅出现在文本节点和摘要字符串中。

## 数据流

1. 后端能力配置继续提供 `auto`、`high` 等枚举。
2. `PromptComposer` 使用共享格式化函数渲染 `<option>` 文本，选项值保持原枚举。
3. 生成请求继续提交原枚举。
4. 当前消息和历史消息在构造参数摘要时使用同一个格式化函数。

## 可访问性与布局

继续使用带标签的原生 `select`，不引入自定义下拉行为。中文文案变长时沿用参数栏现有的非压缩和自动换行规则。

## 测试

- 单元测试覆盖全部映射和未知值回退。
- 组件测试验证三个下拉框显示中文，但触发的更新事件仍携带原始值。
- 消息测试验证当前消息和历史消息摘要使用相同中文标签。
- 完整前端测试、类型检查和生产构建必须通过。
