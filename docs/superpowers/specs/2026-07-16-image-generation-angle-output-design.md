# 图片生成多角度输出设计

## 目标

在图片预设的“角度”区域增加“每个角度单独生成一张”复选框，让用户能在一张多视角拼图与多张独立角度图片之间切换。多张独立图片必须显示在同一条助手结果消息中，并确保每个角度描述与对应图片一一匹配。

## 交互设计

- 复选框位于图片预设弹窗的“角度”选项下方，仅在选择至少两个角度时显示。
- 文案为“每个角度单独生成一张”。
- 默认不勾选。
- 未勾选时，提示用户将生成一张包含所选角度的多视角拼图。
- 勾选时，提示用户将为每个所选角度生成一张图片，结果会显示在同一条消息中。
- 清空预设时同时恢复为未勾选状态。

## 请求行为

### 未勾选

- 只提交一个生成任务，`output_count` 固定为 `1`。
- 不发送 `variants`。
- 提示词包含全部所选角度，并要求在同一张图片中按所选顺序排列为多视角拼图。
- 提示词明确每个分区只对应一个角度，避免角度与画面位置不一致。

### 已勾选

- 只提交一个顶层生成任务。
- 每个所选角度映射为一个 `variant`，其中包含稳定的角度标题和只描述该角度的提示词。
- 每个 `variant` 的输出数量为一张。
- 后端沿用已有 `variants` 能力，将每个结果写入同一历史记录，并在图片结果上保留 `variant_label`。

## 结果展示

- 一次提交只创建一条等待中的助手消息。
- 任务完成后，用一条助手结果消息替换等待消息。
- 独立角度图片在该消息的同一个图片网格中展示。
- 每张图片卡片优先显示 `variant_label`，例如“正面”“背面”“左侧”，作为图片与角度描述的确定性对应关系。
- 图片顺序与用户选择角度的顺序一致。
- 历史记录重新加载时仍投影为一条用户消息和一条助手结果消息，并恢复各图片角度标题。

## 错误处理

- 整个任务失败或取消时，沿用现有失败或取消消息。
- 某个子角度失败时，后端应保留已成功生成的角度图片，并在同一结果消息中说明失败的角度；不得把成功结果拆成多条消息或全部丢弃。
- 返回图片缺少 `variant_label` 时，按请求中的 `variants` 顺序补充标签；无法可靠对应时显示通用图片标题，不猜测角度。

## 状态与兼容性

- `ImagePresetSelection` 增加布尔字段保存独立角度选择；默认值为 `false`。
- 旧历史记录和未包含该字段的状态按 `false` 处理。
- 普通多图数量选择不改变现有行为；本功能只在选择至少两个角度时接管输出组织方式。
- 单个角度选择不显示该复选框，并按普通单图生成处理。

## 测试范围

- 组件测试覆盖复选框显示条件、默认状态、切换事件、清空行为和说明文案。
- composable 测试覆盖未勾选时单张拼图请求、勾选时单请求多 `variants`、单条等待消息和单条结果消息。
- 投影测试覆盖历史记录中的多张 `variant_label` 图片恢复为同一条结果消息。
- 后端测试覆盖 `variants` 顺序与标签保存，以及部分子任务失败时保留成功结果。
- 构建和现有图片生成插件测试必须全部通过。
